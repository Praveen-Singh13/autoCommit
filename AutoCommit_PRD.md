# 🤖 AutoCommit — Intelligent Commit Message Generator

> Automatically generate clean, professional, human-like Git commit messages — powered by AI.

----

## 📌 Product Overview

**AutoCommit** is a VS Code extension that automatically generates meaningful commit messages when a user closes VS Code or manually triggers the command.

**How it works:**

1. Detects uncommitted changes
2. Fetches `git diff` details via shell commands
3. Sends changes to an AI API using Python (`requests`)
4. Generates a meaningful commit message
5. Automatically commits the code

---

## 🚨 Problem Statement

Developers often write poor commit messages, skip commits, or use generic messages like:

- `"fix"`
- `"update"`
- `"changes"`

This leads to:

- Poor project history
- Hard debugging
- Unclear collaboration logs

**AutoCommit solves this by automating commit generation intelligently.**

## ✨ Core Features

### Feature 1 — Detect Changes

**Triggered when:**
- VS Code closes, OR
- Manual command is executed

**Flow:**

```bash
git status
git diff
git diff --staged
```

**Purpose:** Check if changes exist and fetch actual modifications.

---

### Feature 2 — Fetch Git Changes via Shell

Use shell scripting inside Python.

**Commands used:**

```bash
git status --porcelain
git diff
git diff --cached
git branch --show-current
```

**Collects:**
- Modified files
- Added files
- Deleted files
- Diff content

---

### Feature 3 — AI Commit Message Generator

**Uses:** Python `requests` + AI API (OpenAI / Claude / local LLM)

**Input:**
- Git diff content
- Changed file names
- Branch name

**Output:**

```
feat: add login validation logic

- Added password validation
- Improved error handling
```

---

### Feature 4 — Auto Commit Execution

```bash
git add .
git commit -m "generated message"
# Optional:
git push
```

---

### Feature 5 — VS Code Extension Trigger

Minimal JavaScript required. Triggers on:
- VS Code close
- Command palette action

## 🛠️ Tech Stack

| Layer | Technology | Purpose |
|---|---|---|
| Core Logic | Python | Git commands, API calls, commit execution |
| API Calling | `requests` | Send diff to AI, receive commit message |
| Git Interaction | `subprocess` + shell | Run git commands |
| VS Code Layer | JavaScript (minimal) | Trigger Python script, lifecycle events |

### API Call Example

```python
import requests

response = requests.post(
    API_URL,
    headers={"Authorization": f"Bearer {API_KEY}"},
    json={
        "diff": git_diff
    }
)
```

## 🔄 Detailed Workflow

### Step-by-Step Flow

**Step 1 — Trigger Event**
- VS Code closes, OR
- Command is executed

**Step 2 — Detect Changes**

```bash
git status --porcelain
```

> If empty → Exit process

**Step 3 — Fetch Git Diff**

```bash
git diff
git diff --cached
```

> Store: `diff_data`

**Step 4 — Send API Request**

```json
// Request
{
  "diff": "code changes",
  "branch": "feature/login"
}

// Response
{
  "message": "feat: add login validation"
}
```

**Step 5 — Execute Commit**

```bash
git add .
git commit -m "generated message"
```

## 🔌 API Design
